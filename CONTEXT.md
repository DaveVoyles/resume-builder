# Project Glossary

Domain terms, models, and vocabulary this project uses consistently — in
code, comments, commit messages, and conversation. See AGENTS.md
§Engineering Principles ("Ubiquitous language").

| Term | Definition | Notes |
|---|---|---|
| contact | A person-level entity tracked independently of any specific role — a referral, former colleague, recruiter, or informational-interview contact from a job search. Optionally linked to one or more tracked roles, never owned by one. | `contacts.json`; see docs/design/0004. Has its own status enum (identified → reached-out → responded → meeting-scheduled → met → referred → no-response → dormant) that drives an auto-generated `nextAction`, mirroring how `role.application.status` already does for roles. |
| de-AI lint | A deterministic style lint over resume/cover-letter config text — buzzword/cliché wordlist plus sentence-uniformity and repetition heuristics — that flags AI-flavored prose with advisory warnings; never blocks a render, uses no ML. | Introduced by design plan 0005 (D7); concept adapted from scottgal/lucidRESUME's AI-detection scorer, minus its ML models. Rewrite guidance lives in the tailor playbook (D8). |
| debrief | A general Q&A feedback loop: capture a question, the candidate's answer, a sentiment rating, and the AI's proposed improved answer for next time. Applies to any agent-driven Q&A context (grill intake, real interviews, study-guide prep, tailor conversations), not one lifecycle stage. | Entries live in `feedback.jsonl` in the candidate workspace; captured via `docs/playbooks/debrief.md`. Introduced by design plan 0002 (D1). |
| gap taxonomy | The four-type classification of a missing job-posting keyword: PresentationGap (skill held, just reworded/absent from the resume), WeakEvidence (some support, needs strengthening), AdjacentSkill (a related skill could bridge it), TrueGap (genuinely not held). Each gap carries a recommended action. | Introduced by design plan 0005 (D5/D6); adapted from scottgal/lucidRESUME. Classification is agent/playbook work; the CLI validates and renders the advisory `gap-report`. |
| ghosted | A terminal application status for a role whose employer went silent — no response after applying or mid-process. Only ever set manually via `set-status`; staleness detection may suggest it but no code path auto-transitions to it. | Joins the `set-status` enum + nextAction rule table in design plan 0005 (D1); adapted from scottgal/lucidRESUME's pipeline stages. |
| keyword coverage | The percentage of agent-extracted job-posting keywords that appear (case-insensitive substring match) somewhere in a resume config's claim-bearing text (summary, bullets, skills), plus the resulting present/missing lists. | Introduced by the ATS keyword-match scoring feature (docs/design/0004). Always advisory, never blocking; never persisted to a role or the tracker — a point-in-time check, not a tracked field. |
| lead | A prospective role discovered during search, pre-application, kept as a plain agent-maintained record with a fit rating and notes until promoted into tracked roles. | Lives in `leads.json` in the candidate workspace; promoted via the existing `add-role` command. Introduced by design plan 0001 (D6). |
| nextAction | The auto-generated `{type, dueDate}` follow-up a tracked entity carries after a status transition — what to do next, and by when. | `src/core/role-view.js`, `src/cli/commands/set-status.js` (roles); reused verbatim (not reinvented) for the new `contact` entity in plan 0004. |
| playbook | A vendor-neutral markdown instruction doc that any terminal agent follows to do the semantic half of a workflow, handing schema-validated JSON to the deterministic CLI. | Lives in `docs/playbooks/`. The agent/CLI contract is [ADR 0001](docs/decisions/0001-agent-operated-cli.md). |
| semantic compression | The explicit tailor-playbook step of selecting, from all N profile roles, the M relevant to a specific job description — each surviving role justified in one line against the JD — before drafting resume content. | Introduced by design plan 0005 (D8); adapted from scottgal/lucidRESUME. Playbook-only; no CLI involvement. |
| stale | A computed, render-time flag on a tracked role whose days-since-last-touch exceeds the threshold for its status bucket; carries days-since-touch and a suggested nextAction. Never persisted, never mutates state — recomputed on every tracker build. | Introduced by design plan 0005 (D2/D3); thresholds have code defaults, overridable via workspace preferences. Adapted from scottgal/lucidRESUME's stale detection. |
| tailor | The one-pass workflow that turns a drafted, schema-conformant resume config into a validated, evidence-audited, rendered DOCX plus a tracked role — landed with a not-yet-applied `application.status` so a human reviews the resume before it's sent. | `src/cli/commands/tailor.js`; playbook at `docs/playbooks/tailor.md`. Composes D2's `validateResumeConfig`/`renderResumeConfig`, D3's `auditResumeConfig`, and D6/D7's `add-role`/`set-status` rather than reimplementing any of them. Introduced by design plan 0001 (D4). |
